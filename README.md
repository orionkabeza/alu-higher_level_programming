# Python - Hello, World

This is the first Python project in the ALU Higher Level Programming curriculum. It introduces basic Python programming concepts including running Python scripts, printing text and variables, working with strings, and understanding indexing and slicing.

## Project Overview

- **Author**: Guillaume
- **Weight**: 1
- **Course**: Introduction to Programming and Databases

## Learning Objectives

- Understand why Python programming is awesome
- Learn about Python's creator (Guido van Rossum) and the origin of the name "Python"
- Understand the Zen of Python
- How to use the Python interpreter
- How to print text and variables using print
- How to use strings
- Understanding indexing and slicing in Python
- PEP8 coding style and how to check code with pycodestyle

## Requirements

### Python Scripts
- Interpreted/compiled on Ubuntu 20.04 LTS using Python 3 (version 3.8.5)
- All files should end with a new line
- First line must be exactly `#!/usr/bin/python3`
- Must use pycodestyle (version 2.7.*)
- All files must be executable
- Code length will be tested using wc

### Shell Scripts
- Exactly two lines long
- First line must be exactly `#!/bin/bash`
- All files must be executable

## Tasks

| # | Task | Description |
|---|------|-------------|
| 0 | Run Python file | Shell script that runs a Python script |
| 1 | Run inline | Shell script that runs Python code from environment variable |
| 2 | Hello, print | Print "Programming is like building a multilingual puzzle" |
| 3 | Print integer | Print an integer with "Battery street" |
| 4 | Print float | Print a float with precision of 2 digits |
| 5 | Print string | Print a string 3 times followed by its first 9 characters |
| 6 | Play with strings | Concatenate strings to print "Welcome to Holberton School!" |

| 8 | Create a new sentence | Print "object-oriented programming with Python" |
| 9 | Easter Egg | Print "The Zen of Python" by Tim Peters |

## Running the Files

```
bash
# Make a Python file executable
chmod +x filename.py

# Make a shell script executable
chmod +x filename

# Run a Python script
./filename.py

# Run a shell script
./filename
```

## The Zen of Python

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
Special cases aren't special enough to break the rules.
Although practicality beats purity.
Errors should never pass silently.
Unless explicitly silenced.
In the face of ambiguity, refuse the temptation to guess.
There should be one-- and preferably only one --obvious way to do it.
Although that way may not be obvious at first unless you're Dutch.
Now is better than never.
Although never is often better than *right* now.
If the implementation is hard to explain, it's a bad idea.
If the implementation is easy to explain, it may be a good idea.
Namespaces are one honking great idea -- let's do more of those!
